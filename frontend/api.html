ephemeral api

v1 · public · .website only · home

Upload audio, get a one-time playback link. When the recipient plays the link once, the audio is deleted and every endpoint referring to the token returns 404. There is no way to undelete a whisper. 404 is indistinguishable from "never existed."

This API is public — no keys, no auth. Browser-based CORS is restricted to the .website family of first-party layers (allowlist: ephemeral.website, appreciation.website, confession.website). Server-to-server calls work from anywhere. It is intended to be built upon.

Stability

These routes are v1 and will not change in a breaking way. New fields may be added to responses; existing fields will not be removed or renamed. Breaking changes would land under /api/v2/* with a deprecation period on v1.

Endpoints

POST /api/upload

Create a whisper. Returns a token, a pre-signed S3 PUT URL, and the s3_key used for storage.

// request
{
  "slug": "optional-custom-slug",
  "note": "optional short note (≤ 280 chars)",
  "waveform": "optional comma-separated peaks 0-100",
  "content_type": "audio/mpeg",
  "s3_key": "audio/<uuid>  (optional — see upsert)"
}

// response  201 Created
{
  "token": "my-slug-or-uuid",
  "url": "/my-slug-or-uuid",
  "upload_url": "https://s3.amazonaws.com/... (presigned PUT)",
  "s3_key": "audio/<uuid>"
}

Errors: 400 invalid body / format, 409 slug taken, 500 store or S3 error.

Upsert semantics. If the slug already exists and you supply the same s3_key the server returned on the original create, the existing row is updated in place (note and waveform) instead of colliding. This is the authentication primitive: only the original creator holds the s3_key.

GET /api/check/{token}

Check whether a token still exists. Safe to call repeatedly — does not burn the token.

// response 200  (token exists)
{
  "exists": "true",
  "note": "optional note",
  "waveform": "10,20,30,...",
  "stream_url": "https://s3... (presigned, short-lived)"
}

// response 200  (token does not exist)
{ "exists": "false" }

exists is a string, not a boolean.

POST /api/session

Burn a token and open a playback session. Atomic — the second call on the same token returns 404.

// request
{ "token": "my-slug-or-uuid" }

// response 201
{
  "session_id": "<uuid>",
  "pause_timeout": 15,
  "note": "optional note"
}

GET /api/stream/{session_id}

Stream the audio bytes for an active session. Raw audio with Content-Type header. Supports HTTP range requests.

<audio src="https://ephemeral.website/api/stream/<session_id>" controls></audio>

POST /api/heartbeat/{session_id}

Reset the pause timer. Call every ~5s while audio is playing. Returns 404 if the session expired.

POST /api/complete/{session_id}

Mark session complete. Deletes the S3 object. Call on the ended event of the audio element.

Typical flows

Create and share a whisper

1. POST /api/upload     { slug, content_type }  →  { token, upload_url, s3_key }
2. PUT  upload_url      (audio bytes)           →  200
3. Share https://ephemeral.website/<token>

Build a layer that sends to a handle

A layer (e.g. appreciation.website) can delegate all audio storage to ephemeral and track its own handle → token mapping:

On upload:
1. POST /api/upload     { slug, content_type }  →  { token, upload_url, s3_key }
2. PUT  upload_url      (audio bytes)
3. Save (recipient_handle, token) in layer's own DB

On inbox view:
4. GET /api/check/{token}  →  confirm whisper still exists
5. Redirect user to https://ephemeral.website/<token> for playback
   (or embed the core player)

Layers never touch audio bytes after upload. The core's one-listen rule is enforced regardless of how many layers reference it.

2-phase reservation for instant share UX

On file select:
1. POST /api/upload   { content_type }                     →  { token0, upload_url, s3_key }
2. PUT  upload_url    (start audio upload in background)

On slug blur:
3. POST /api/upload   { slug, content_type, s3_key }       →  { token: slug }  (reserves)

On send:
4. POST /api/upload   { slug, note, waveform, ..., s3_key } →  upserts note+waveform
5. Show share page immediately — no waiting.

Example

curl -X POST https://ephemeral.website/api/upload \
  -H 'Content-Type: application/json' \
  -d '{"slug":"just-for-you","content_type":"audio/mpeg"}'
# → { "token":"just-for-you", "upload_url":"https://...", "s3_key":"audio/..." }

curl -X PUT "$UPLOAD_URL" \
  -H 'Content-Type: audio/mpeg' \
  --data-binary @voice-memo.mp3
# → 200

# Share: https://ephemeral.website/just-for-you
